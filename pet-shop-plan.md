
# 專案規劃書：寵物食品銷售平台 (BaanPets 風格參考)

## 1. 專案目標
開發一個響應式寵物食品銷售平台，具備商品瀏覽、購物車、會員系統與後台訂單管理功能。目標風格為簡潔、清新、親和力強，參考 `baanpets.com` 之視覺與互動體驗。

## 2. 技術堆疊
- **後端框架：** Laravel 12 (PHP 8.2+)
- **資料庫：** MySQL
- **前端呈現：** Blade Templating + Tailwind CSS (快速實現現代簡潔 UI) + Alpine.js (輕量互動)
- **環境：** Local Development (Valet/Docker)

## 3. 核心功能需求 (MVP - 兩週版)
- **使用者端：**
    - 首頁：Banner, 精選商品推薦, 分類導覽。
    - 商品頁：商品清單 (篩選/排序), 商品詳細頁。
    - 購物車：新增、修改數量、結帳。
    - 會員：登入、註冊、個人訂單歷史。
- **管理端 (Admin)：**
    - 商品管理 (CRUD)。
    - 訂單狀態管理。

## 4. 時程規劃 (2026/03/15 - 2026/03/29)

### 第一週：基礎架構與核心功能
- **Day 1-2：架構與環境設定**
    - Laravel 12 安裝, DB 連線設定, 專案結構初始化。
    - 設計資料庫 Schema (Users, Products, Categories, Orders)。
- **Day 3-5：會員系統與商品邏輯**
    - Laravel Breeze 實作認證系統 (Auth)。
    - 開發 Model/Controller/View，實作商品 CRUD。
- **Day 6-7：前端 UI 實作**
    - 導入 Tailwind CSS，根據參考網址風格設計樣式。
    - 完成首頁與商品列表頁。

### 第二週：購物流程、後台與優化
- **Day 8-10：購物車與訂單系統**
    - 實作 Session-based 購物車。
    - 訂單結帳邏輯與資料寫入。
- **Day 11-12：管理後台與測試**
    - 開發簡單的管理儀表板 (商品上下架/訂單檢視)。
    - 完成功能驗收測試 (測試登入、購物、結帳流程)。
- **Day 13-14：細節調優與部署準備**
    - 響應式優化 (Mobile-first)。
    - 錯誤處理與表單驗證優化。

## 5. 驗收標準
- 成功執行 `php artisan migrate`，資料表建立正常。
- 測試帳號可完成從商品瀏覽到結帳的全流程。
- 網頁於行動裝置與桌機皆可正常瀏覽 (RWD)。
- 後台可正確新增商品並於前台顯示。
